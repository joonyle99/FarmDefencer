# 타입: 커밋의 유형은 제목 내에 포함되며 다음 중 하나를 선택
#
# - [Feat] : 새로운 게임 기능 추가
# - [Fix] : 게임 내에서 발견된 버그 수정
# - [Refactor] : 코드 구조 개선 (기능 변화 없음)
# - [Test] : 테스트 코드 추가 혹은 테스트 관련 수정
# - [Style] : 코드 스타일 수정 (세미콜론, 줄바꿈, 띄어쓰기, 들여쓰기 등)
# - [Docs] : 문서 수정 (README.md, 코드 주석, 기술 문서 등)
# - [Chore] : 빌드 스크립트, 패키지 설정 등 코드 외 작업
################

################
# 제목(Subject)
#
# - 제목은 한 문장으로 간결하게 작성 (최대 50자)
# - 영문 표기 시 첫글자는 대문자로, 명령형으로 작성 (과거시제 금지)
# - 불필요한 특수기호, 마침표 생략
# - 제목에 타입을 포함하여 어떤 변경인지 명확하게 표현
# - 커밋 메시지에서 변경 사항이 명확하게 드러나도록 작성
################

################
# 본문(Body)
#
# - 본문은 제목과 빈 줄로 구분
# - '어떻게(How)' 보다는 '무엇(What)'과 '왜(Why)'에 중점
# - 구체적인 이유나 해결한 문제의 맥락을 설명
# - 필요 시 72자 이하로 줄바꿈하여 작성
################
